JumpMind Commerce Security Matrix

JumpMind Commerce supports the idea that users belong to one or more workgroups.

Each workground is assigned to one or more permisison id. At various points throughout the the application's business flows, the system will check if the current user has the specific permission id or not. If the current user does not have permission and the permission id is overridable, then the system will prompt for a manager override so a more privileged user can approve the operation.

Workgroups can be assigned to specific permission ids, or workgroups can be assigned to wildcarded permission ids. For example, an administrator could create a workground of "sellers" and grant the sellers permission id of sell.*
